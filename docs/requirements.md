# DogWalk 프로젝트 요구사항 정의서 (MVP 기준)

## 1. 프로젝트 개요
DogWalk는 반려견 대리산책 매칭/수행, 사용자의 셀프 산책 기록, 크레딧 리워드, 쇼핑몰 연계를 제공하는 서비스다.

## 2. 이해관계자 및 사용자 타입
- 견주(Owner): 반려견 정보를 등록하고 대리산책을 요청하거나 셀프 산책을 기록한다.
- 워커(Walker): 자격 검증 후 대리산책 요청을 수락/수행한다.
- 관리자(Admin): 회원 심사, 운영 모니터링, 결제/환불/정산을 관리한다.

## 3. 핵심 기능 요구사항

### 3.1 대리산책 서비스
1. 견주는 예약 시간, 반려견, 요청사항을 입력해 대리산책을 요청할 수 있어야 한다.
2. 워커는 요청을 수락/거절할 수 있어야 한다.
3. 산책 진행 중 견주는 워커의 실시간 위치를 지도로 확인할 수 있어야 한다.
4. 산책 종료 후 시간, 거리, 평균 속도, 사진 리포트를 확인할 수 있어야 한다.
5. 결제는 앱에서 완료되어야 한다(카드/간편결제/오프라인 카드단말 옵션 포함).

### 3.2 셀프 산책 기능
1. 견주는 시작/종료 버튼으로 셀프 산책을 기록할 수 있어야 한다.
2. 종료 시 시간/거리/평균속도 요약이 표시되어야 한다.
3. 거리/시간 기준으로 크레딧을 자동 적립해야 한다.
4. 산책 기록에 사진 및 메모를 남길 수 있어야 한다.

### 3.3 쇼핑몰/크레딧
1. 제휴 상품 목록을 조회하고 주문/결제가 가능해야 한다.
2. 결제 시 보유 크레딧을 사용할 수 있어야 한다.
3. 크레딧은 원장(ledger) 기준으로 적립/사용/소멸 이력이 남아야 한다.

### 3.4 회원가입 및 검증
1. 견주 가입 시 본인정보 + 반려견정보(동물등록번호 포함 가능)를 입력해야 한다.
2. 워커 가입 시 자격증 업로드 및 관리자 승인이 필요하다.
3. 프로필/반려견 사진 등록이 가능해야 한다.

### 3.5 관리자 웹
1. 워커 자격 검수(승인/반려), 회원 제재, 분쟁 관리 기능이 필요하다.
2. 결제 조회, 환불 처리, 정산 배치 실행이 가능해야 한다.
3. 운영 대시보드(진행 중 산책, KPI)를 제공해야 한다.

## 4. 비기능 요구사항
- 보안: 개인정보/위치정보 최소수집, 암호화 저장, 접근권한 통제(RBAC)
- 신뢰성: 위치 로그 유실 최소화(오프라인 재전송)
- 성능: 실시간 위치 갱신 지연 최소화
- 감사추적: 주요 상태 변경/정산/환불 이력 기록

## 5. MVP 범위
- 필수: 회원가입, 반려견 등록, 대리산책 예약/수행, 실시간 위치, 결제, 산책 리포트, 셀프 산책 기록, 관리자 심사/환불/정산 최소기능
- 후속: 추천 매칭 고도화, 고급 통계, 구독제, 커머스 고도화
